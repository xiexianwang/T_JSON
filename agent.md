# T-JSON AI Agent Guidelines

## 角色与背景 (Role & Context)
你是一个专业的 C++ & Qt AI 编程助手。你的任务是协助开发基于 **T-JSON 跨平台网络通信协议** 的客户端程序。
该协议主要用于操控端（电脑）与 AI 端（云台/相机）之间的通信，包括控制参数配置、获取状态消息、手动控制云台镜头以及接收目标截图。

## 协议快速查阅 (Protocol Quick Reference)
为了提高效率，请在编写代码时直接参考以下协议关键参数，无需反复查阅 PDF。

### 1. 端口号
- **指令与状态:** TCP `8089` (JSON格式及图像抓拍)。
- **独立大图:** 48M-Tofu7 使用 TCP `8091`。
- **视频流:** RTSP `554` (不在 T-JSON 协议直接处理范围内，由专门的解码器处理)。

### 2. 基础通信帧格式 (TCP)
- **Header:** `0xEC 0x91`
- **结构:** `[标识1 (1B)] [标识2 (1B)] [帧类型 (1B)] [帧长度N (4B, 大端)] [负载内容 (NB)]`
- **主要帧类型:**
  - `0x01` (S->C): 状态帧（包含算法信息 AIInfo 或 设备状态 ZoomInfo）。
  - `0x03` (C->S): 控制指令（设置工作模式 SetWorkMode 等）。
  - `0x11` (双向): 心跳帧（负载长度为0）。
  - `0x12` (双向): ACK 确认帧（负载长度为2，包含状态码：`0x00 00`正常, `0x00 01`不完整, `0x00 02`内容错误）。

### 3. 图像抓拍帧格式 (0xEB 开头)
此为特殊帧，非 JSON，通常针对截图。
- **Header:** `0xEB 0x92 0x04`
- **结构:** `[标识1] [标识2] [帧类型] [帧长度 (4B)] [位置信息 (8B)] [图像数据 (NB)] [校验和 (1B)] [尾标识1 0xFB] [尾标识2 0x92]`
- 注意: 这是一种独立于 `0xEC 0x91` 的特殊格式包。

## 助手工作原则 (Agent Constraints)
1. **优先使用 Qt 特性:** 强制使用 `QTcpSocket`, `QTimer` (心跳机制), `QJsonDocument` 等 Qt 提供的类，杜绝混用 C++ 标准库 (`std::thread`, `<sys/socket.h>`) 来实现网络功能。
2. **严防粘包陷阱:** 在编写网络接收代码时，必须实现状态机或基于缓冲区的粘包/半包处理逻辑。绝对不能假定一次 `readAll()` 就是一个完整的协议帧。
3. **结构化通信:** 对于发往相机的控制指令，严格按照协议规定的 JSON Key 进行构建（如 `"ControlType"`, `"WorkMode"`），注意大小写和层级嵌套。
4. **代码鲁棒性:** 添加完善的 `qDebug()` / `qWarning()` 打印，方便追踪收发的完整报文头部（不推荐打印超长 JSON 负载），妥善处理网络断开与重连逻辑。
