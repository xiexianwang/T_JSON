# T-JSON UI & Layout Constraints (Customer-Facing Version)

## 1. 产品定位与框架选择
- **产品定位**: 本软件交付给**终端客户**使用，而非开发人员的底层调试工具。界面必须干净、直观、业务导向。**严禁**在 UI 面板中暴露底层网络协议的原始报文（如 RX/TX HEX 数据流、原始 JSON 字符串）。
- **UI 框架**: 强制使用 **Qt Widgets (C++)** 进行开发。主窗体必须继承自 `QMainWindow`。

## 2. 整体布局规则 (Layout Rules)
- 采用 **固定全景布局 (Fixed Dashboard Architecture)**：彻底放弃多页签和滚动条。在 `CentralWidget` 中进行明确的静态分块，确保用户打开软件即可在同一屏幕内一览无余所有的操作与状态。
- **布局分块**：
  - **顶部栏 (Top Bar)**：设备连接配置 (IP, 端口) 与全局连接状态，水平布局 (`QHBoxLayout`)。
  - **中部核心区 (Middle Area)**：
    - **左侧 (Display Area)**：视频流与目标图像渲染区，占据最大化比例。支持比例锁定 (16:9) 与黑边坐标逆映射。
    - **右侧 (Control Panel)**：放置**纯主动控制**类组件（AI工作模式、算法切换、云台控制盘、预置位、经纬度下发），垂直紧凑布局，不需要滚动条。
  - **底部仪表盘 (Bottom Dashboard)**：水平平铺 5 个等宽的状态信息组，专门用于展示**被动接收/上报**的数据：
    1. 云台状态 (PTZ - 角度、经纬高、模式、距离)
    2. 镜头状态 (Lens - 双光的倍率、焦距、视场角、聚焦)
    3. 识别上报 (Identification Report - 展示多目标列表)
    4. 跟踪上报 (Tracking Report - 展示单目标锁定状态)
    5. 图像与系统参数 (Image Params - 包含获取按钮与参数展示)

## 3. 关键组件强制约束 (Component Enforcements)
- **业务数据展示 (底部面板)**：
  - “识别上报”模块推荐使用 `QTableWidget`，以便清晰列出当前画面中识别出的多个目标 (ID、类别、坐标)。
  - 其余状态监控均采用 `QFormLayout` 配合 `QLineEdit (ReadOnly)` 或 `QLabel` 进行键值对展示。
- **参数安全边界**：向用户开放的参数设置必须提供输入校验，`QSpinBox` 和 `QDoubleSpinBox` 必须设定符合设备物理限制的最大/最小值，防止客户输入破坏性参数。

## 4. 视觉与交互风格 (Visual & UX Theme)
- **视觉焦点**：全界面采用深色扁平化 QSS 样式设计，降低周遭控件对视觉的干扰，使用户的注意力集中在中央的视频监控画面上。
- **现代扁平化**：摒弃传统操作系统的 3D 凸起边框外观。按钮等可交互元素必须具备 Hover、Pressed 和 Disabled 状态的颜色反馈。
- **操作反馈与线程安全**：网络重连等耗时操作必须在后台处理，绝不允许阻塞 UI 线程导致界面“假死”。成功/断开连接必须在界面给予客户直观反馈（如按钮变色、状态栏提示）。

## 5. 画面渲染与框选坐标映射 (Coordinate & Rendering Rules)
- **等比渲染与黑边 (Letterboxing)**：为了保证目标不被拉伸变形，画面控件必须强制保持原始长宽比（通常为 16:9）。当主窗体拉伸时，渲染图像必须采用等比缩放（`Qt::KeepAspectRatio`），多余部分留黑边处理。
- **UI 坐标到真实坐标的逆映射**：**严禁直接将 UI 控件的鼠标坐标作为目标坐标下发给设备！** 在进行“框选跟踪”等点选操作时，必须在 C++ 代码中实现严格的坐标映射逻辑：
  1. 获取鼠标在控件上的 `(X_ui, Y_ui)`。
  2. 减去因为等比缩放产生的黑边偏移量 `(Offset_X, Offset_Y)`。
  3. 除以当前的渲染缩放系数 `Scale`。
  4. 换算回相机原始分辨率（如 1920x1080）下的绝对坐标 `(X_real, Y_real)` 后，再通过 JSON 发送给 AI 设备。
