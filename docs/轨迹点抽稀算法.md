# 轨迹点抽稀算法

## 概述

设备每 40ms 上报一次 `AIInfo` 帧（25fps），包含锁定目标的像素坐标。若全部绘制到地图，一分钟产生 1500 个点，浏览器性能不可接受。本算法在 C++ 侧做**智能抽稀**，前端做**FIFO 上限控制**，在视觉连续性和性能间取得平衡。

---

## 一、阈值常量

| 参数 | 值 | 作用 |
|------|-----|------|
| `TRK_MIN_DIST_M` | 3.0 m | 防抖死区：小于此距离直接丢弃 |
| `TRK_MAX_DIST_M` | 20.0 m | 长距离抽稀：超出此距离强制打点 |
| `TRK_HEADING_DIFF_DEG` | 15.0 ° | 航向角偏转阈值 |
| `TRK_HEARTBEAT_MS` | 2500 ms | 心跳间隔：超时强制打点 |

### 2.3 关键代码

```cpp
static bool shouldPlotTrackPoint(double newLat, double newLon,
                                  double plotLat, double plotLon,
                                  double plotHeading, const QDateTime& plotTime)
{
    if (plotHeading > 360) return true; // 首次绘制

    double dist = haversineDistance(plotLat, plotLon, newLat, newLon);

    // 防抖死区：漂移直接丢弃
    if (dist < TRK_MIN_DIST_M) return false;

    // 长距离抽稀
    if (dist > TRK_MAX_DIST_M) return true;

    // 心跳兜底
    if (plotTime.isValid()) {
        qint64 elapsed = plotTime.msecsTo(QDateTime::currentDateTime());
        if (elapsed >= TRK_HEARTBEAT_MS) return true;
    }

    // 航向角变化
    double head = bearing(plotLat, plotLon, newLat, newLon);
    double diff = qAbs(head - plotHeading);
    if (diff > 180.0) diff = 360.0 - diff; // 归一化到 [0, 180]
    if (diff >= TRK_HEADING_DIFF_DEG) return true;

    return false;
}
```

## 三、航向角计算

### 3.1 Bearing 公式

计算从点 A `(lat1, lon1)` 到点 B `(lat2, lon2)` 的方位角（正北为 0°，顺时针方向）：

```
lat1R = lat1 → rad
lat2R = lat2 → rad
lon1R = lon1 → rad
lon2R = lon2 → rad
dLon = lon2R - lon1R

y = sin(dLon) × cos(lat2R)
x = cos(lat1R) × sin(lat2R) - sin(lat1R) × cos(lat2R) × cos(dLon)

bearing = atan2(y, x) → deg
if (bearing < 0) bearing += 360
```

位置：`src/mainwindow.cpp:1056-1068`

### 3.2 角度差归一化

航向角是环形量（0° = 360°），不能直接相减：

```cpp
double diff = abs(newHeading - plotHeading);
if (diff > 180.0) diff = 360.0 - diff;
```

**例 1**：新航向 = 10°，旧航向 = 350°
- 差值 = |10 - 350| = 340
- 340 > 180 → diff = 360 - 340 = **20°** ✓（实际偏转 20°）

**例 2**：新航向 = 270°，旧航向 = 90°
- 差值 = |270 - 90| = **180°** ✓（完全掉头）

### 3.3 `TrackState` 结构体

```cpp
struct TrackState {
    QString id;                    // 当前跟踪目标 ID
    double lat = 0, lon = 0;       // 最后已知位置
    int cls = 0;                   // 最后 Class
    QDateTime lostSince;           // 首次失锁时间（空 = 锁定中）

    QDateTime prevTime;            // 上一个帧时间

    // 抽稀状态
    double plotLat = 0, plotLon = 0;  // 上次实际绘制到地图的点
    double plotHeading = 999;         // 上次绘制段航向角，999=未初始化
    QDateTime plotTime;               // 上次绘制时间（心跳用）
};
```

位置：`src/mainwindow.h:114-127`

**关键区分**：
- `prevLat/prevLon` — **每帧都更新**，用于速度计算（两帧间距离/时间）
- `plotLat/plotLon` — **只在实际画点时更新**，用于抽稀判定

## 四、前端 FIFO 上限控制

### 4.1 代码

```javascript
var MAX_TRACK_PTS = 2000;   // 每条轨迹最大点数

function jsAddTrackPoint(trackId, lat, lon) {
    var ll = [lat, lon];
    if (!trackPolylines[trackId]) {
        trackPolylines[trackId] = L.polyline([], {
            color: '#ff4444', weight: 3, opacity: 0.8,
            smoothFactor: 1.5    // 减少折线渲染顶点数
        }).addTo(trackLayers);
        trackEndMarkers[trackId] = L.marker(ll, {
            icon: dotIcon, zIndexOffset: 500
        }).addTo(trackLayers);
    }

    trackPolylines[trackId].addLatLng(ll);

    // FIFO 上限控制：超出 MAX_TRACK_PTS 时淘汰最早的点
    var latlngs = trackPolylines[trackId].getLatLngs();
    if (latlngs.length > MAX_TRACK_PTS) {
        latlngs.splice(0, latlngs.length - MAX_TRACK_PTS);
        trackPolylines[trackId].setLatLngs(latlngs);
    }

    trackEndMarkers[trackId].setLatLng(ll);
}
```

位置：`resources/map.html:497-537`

### 4.2 `smoothFactor` 的作用

Leaflet 的 `L.Polyline` 支持 Douglas-Peucker 简化。`smoothFactor: 1.5` 表示在低缩放级别时合并临近顶点，高缩放级别恢复细节。值越大简化越激进，适合高密度的轨迹线。

## 五、完整数据流

```
设备 (40ms)
  │
  │  AIInfo 帧 (pixelX, pixelY, distance)
  ▼
pixelToGps()  →  设备 GPS + 云台角度 + 像素偏移 → 目标 GPS
  │
  ▼
shouldPlotTrackPoint()
  │  3m 内?      → ✗ 丢弃
  │  20m 外?     → ✓ 画点
  │  2.5s 超时?  → ✓ 画点
  │  转弯 ≥15°?  → ✓ 画点
  │  其他        → ✗ 丢弃
  │
  ▼   (判定通过)
appendTrackPoint()  → 缓存到 m_cacheTrackPts
  │
  ▼   (200ms 定时器)
flushAiData()  → 批量 runJS("jsAddTrackPoint(...)")
  │
  ▼
前端 FIFO 队列 (max 2000)
  │  addLatLng → 超出 splice 最早点 → setLatLngs
  ▼
Leaflet L.polyline 渲染
```

## 六、场景示例

| 场景 | 目标行为 | 打点频率 | 说明 |
|------|---------|---------|------|
| 悬停 | GPS 漂移 < 3m | 0 点 | 防抖死区全部过滤，不画 |
| 低速直线 (5m/s) | 5m/s × 2500ms = 12.5m | 每 2.5s 一个点 | 心跳兜底，不到 20m 也画 |
| 高速巡航 (50m/s) | 50m/s × 400ms = 20m | 每 400ms 一个点 | 长距离抽稀触发 |
| 急转弯 (航向变 90°) | 偏转超过 15° | 立刻打点 | 航向角判定优先于距离 |
