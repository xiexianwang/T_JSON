# T-JSON 项目开发日志

## 日期：2026年05月24日

### 🎯 核心目标
分析 T-JSON 通信协议规范，完成设备端与客户端的底层 TCP 通信链路搭建、UI 基础交互绑定以及系统架构重构。

### ✅ 本次完成的功能与优化

#### 1. 底层通信框架 (`TJsonClient`)
- **TCP 异步链路建立**：基于 `QTcpSocket` 实现了非阻塞模式的数据收发（默认连接 `192.168.1.200:8089`）。
- **心跳保活机制**：实现 `QTimer` 定时任务，在连接建立后严格按照每 10 秒发送 `0x11` 心跳帧，维持链路长连接。
- **粘包与半包处理**：实现了严格的大端序 (Big-Endian) 包长读取与缓冲区 (`QByteArray`) 截断，支持 `0x01` 状态帧与 `0x04` 图像抓拍帧的精准拆解。
- **防死锁防御机制**：为 JSON 负载（限制 10MB）和 JPEG 负载（限制 50MB）增加最大长度拦截。遇到错位或超大包长时主动丢弃残损帧头并重新寻址，彻底防止了恶意大包导致的系统内存溢出（OOM）和业务死锁。
- **工业级静默重连**：实现了基于指数退避算法的无限期断线重连（2s, 4s, 8s... 最高封顶 60s），抗击网络闪断与设备重启，并实时在界面底部状态栏反馈重连倒数状态。

#### 2. 业务逻辑分离与解耦 (`DeviceController` & `ProtocolBuilder`)
- **透明数据网关**：剥离了具有强烈耦合性的 `sendPtzCommand`，提炼出极其干净的透传网关 `sendTransparentData(serialType, data)`。
- **协议构造器独立**：创建 `ProtocolBuilder` 工具箱，专门封装 `Pelco-D`（包含自动 Checksum 校验和）等基础协议生成，完美符合“开闭原则（OCP）”，为未来对接 VISCA、Pelco-P 预留了空间。

#### 3. 弹窗与参数配置 (`SettingsDialog`)
- **参数设置面板**：使用纯 `.ui` 实现了涵盖“通信与协议配置”与“光学与相机参数”的两大模块弹窗。
- **数据持久化**：使用 `QSettings("Tofu", "T-JSON_Settings")` 完成了针对云台地址、各种通道协议类型、像元尺寸、焦距等数据的本地系统级存储，并支持再次启动应用时自动回显。

#### 4. UI 界面与工程环境修复
- **QSS 主题抽离**：将硬编码的深色扁平化样式彻底抽离为独立的 `style.qss`，并通过 `resources.qrc` 打包进应用程序。
- **乱码与编译修复**：
  - 彻底修复了原 Git 仓库中由于 GBK/UTF-8 转换截断引起的 `mainwindow.ui` 文件 XML 标签损坏（`?/string>`），解决了 `uic` 编译崩溃以及 `Stack corrupted` ABI 错位报错。
  - 在 `CMakeLists.txt` 注入 `/utf-8` 编译指令，并将所有源文件转换为带 BOM 的 UTF-8 编码，消除了各类因 MSVC 本地字符集问题导致的“乱码（Mojibake）”顽疾。

## 日期：2026年05月26日

### ✅ 本次完成的功能与优化

#### 1. 识别/跟踪上报修复
- **ACK 帧处理**：修复 `0x12` ACK 帧被误当作 JSON 帧解析的问题，添加独立分支发射 `ackReceived(quint8)` 信号
- **trackId 修正**：跟踪上报中 `trackId` 曾被赋值状态文本（"锁定中"/"丢失"），改为使用上报 JSON 中 `Object` 内的真实 ID
- **trackAngle 计算**：当 JSON 缺少 Angle 字段时，根据脱靶量 `atan2(dy, dx)` 计算跟踪角度
- **跟踪模式扩展**：从仅支持 `workMode==2` 扩展为 `workMode>=2 && workMode<=4`（框选跟踪/点选跟踪）
- **空对象判断**：将 `count>0` 的守卫条件改为检查 `Object` 非空 (`!isEmpty()`)

#### 2. 图片抓拍功能
- **`onImageSnapped` 实现**：将抓拍 JPEG 保存至 `snapshots/` 目录，文件名包含时间戳

#### 3. 地图功能
- **添加地图组件**：实现 `MapWidget` / `MapBridge` / `MapDialog` 三个类，使用 QWebEngineView + Leaflet 地图
- **Leaflet 内联化**：将 `leaflet.css` 和 `leaflet.js` 内联到 HTML 中，避免 CDN 加载失败
- **顶部栏地图按钮**：在顶部工具栏"断视频"按钮后添加 "🗺 地图" 按钮，弹出地图对话框
- **坐标转换**：实现 `pixelToGps()` 方法，基于航位推算法将图像像素偏移 + 设备 GPS + PTZ 角度转换为目标 GPS
- **设备位置/FOV 显示**：通过 `updateMapDevicePosition()` 根据 ZoomInfo 帧更新设备标记和视场角锥
- **识别目标显示**：通过 `updateMapTargets()` 在地图上显示识别框选标记和跟踪轨迹

#### 4. 地图瓦片源切换到高德地图
- **OSM 被墙**：`tile.openstreetmap.org` 在境内网络不可用（超时），地图白屏
- **迁移至 高德 JS API**：使用高德官方 JS API v2.0（`webapi.amap.com`），无需手动拼接瓦片 URL
- **坐标纠偏**：添加 WGS-84 → GCJ-02 坐标转换，并将所有坐标在显示前进行纠偏
- **移除 Leaflet**：去掉 `leaflet.css` / `leaflet.js` 资源引用，简化 `mapwidget.cpp` 加载逻辑

#### 5. 其他优化
- **可见光默认分辨率**：从 1920×1080 改为 2688×1520（2K）
- **框选跟踪命令**：`onVideoSelection` 区域框选后发送 `SetWorkMode 4` + `P2Track`
- **VideoWidget 比例**：保持 16:9 画面比例
- **ImageSetting 显示**：参数值从数字改为文字描述
